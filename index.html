PROJET DÉTAILLÉ

Borne de Commande Restaurant

Application WPF complète avec architecture MVVM — Janvier 2026 - Mars 2026

🎯 Contexte et Objectifs

Ce projet consiste à développer une borne de commande automatisée pour restaurant, permettant aux clients de commander et payer sans intervention humaine. L'objectif est de réduire les files d'attente, minimiser les erreurs de commande et offrir une expérience moderne et intuitive.

Architecture MVVM

  • Models : Entités métier (Produit, Catégorie, Panier, LigneCommande)
  • ViewModels : Logique de présentation avec INotifyPropertyChanged
  • Views : Interfaces XAML (Veille, Choix, Menu, Panier, Paiement)
  • Services : DataService pour accès MySQL via ADO.NET
  • Helpers : RelayCommand pour implémenter ICommand

Technologies Utilisées

  • C# avec WPF
  • MySQL MySql.Data
  • Pattern MVVM
  • XAML pour interfaces tactiles
  • Windows 10/11

Fonctionnalités réalisées

  • Écran de veille attractif (VeilleWindow)
  • Choix Sur place / À emporter
  • Navigation par catégories et ajout au panier
  • Gestion du panier (quantités, suppression, totaux)
  • Simulation paiement CB/Espèces + n° commande
  • Interface Admin avec authentification

Concept Technique : Pattern MVVM

📦 Model

Entités métier pures (Produit, Panier…) sans logique d'affichage

🔄 ViewModel

Contient la logique, expose les données via propriétés et commandes ICommand

🎨 View

XAML pur, liée au ViewModel via DataBinding sans code-behind métier

Framework Symfony

Le framework PHP professionnel de référence

Pourquoi Symfony ?

Symfony est le framework PHP le plus utilisé en entreprise en France et en Europe. Après avoir travaillé avec du PHP "natif" dans mes projets (site vitrine, ERP), je souhaite monter en compétences vers un framework professionnel qui structure mieux le code et offre des outils puissants.

  • Très demandé sur le marché du travail

Mes Sources Principales

1. Documentation Officielle Symfony (symfony.com/doc)

La référence absolue avec des guides clairs, des exemples de code et les best practices.

2. SymfonyCasts (symfonycasts.com)

Plateforme de tutoriels vidéo créée par des experts Symfony. Parfait pour apprendre avec des cas pratiques concrets.

Ce que j'en retire concrètement

📌 Exemple concret 1 : Doctrine ORM

J'ai appris comment Doctrine permet de manipuler la base de données avec des objets PHP au lieu de requêtes SQL brutes. Ça évite les erreurs SQL et rend le code plus maintenable.

📌 Exemple concret 2 : Système de composants

Symfony est modulaire : Possibiliter d'utiliser un module sans installer tout le framework.

📌 Exemple concret 3 : Console Symfony

J'ai découvert les blocs de commandes qui permettent d'automatiser des tâches (créer un contrôleur, générer une entité...). Ça accélère beaucoup le développement comparé à tout créer manuellement.

Sécurité & Protection des Données (RGPD)

Conformité et bonnes pratiques de sécurité

Ma Certification CNIL 2025

J'ai obtenu la certification RGPD de la CNIL en 2025, ce qui me permet de comprendre les obligations légales lors du développement d'applications manipulant des données personnelles. Cette certification est essentielle car toute application web collecte des données (emails, noms, adresses...).

Mes Sources de Veille RGPD

1. Site officiel CNIL (cnil.fr)

Je consulte régulièrement :

  • Les actualités réglementaires (nouvelles lois, mises à jour)

2. Cybermalveillance.gouv.fr

Alertes sur les failles de sécurité, les cyberattaques récentes et les bonnes pratiques pour protéger les applications web (injections SQL, XSS, CSRF...).
